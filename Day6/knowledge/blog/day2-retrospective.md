# Day2 复盘

## 今天完成了什么

1. 在 Day1 的最小 CLI 架构基础上，新建了 Day2 独立项目目录。
2. 把自由文本问答改成了固定 JSON 输出流程。
3. 用 Zod 对模型输出做了解析和结构校验。
4. 增加了校验失败自动重试逻辑，并补齐了 `TASKS.md` 和 `README.md`。

## 今天最大的收获

- 模型返回了文本，不等于这个结果已经可用。
- 只有经过 `JSON.parse` 和 Schema 校验后，结果才适合继续给前端或后续流程消费。
- 自动重试不是“网络重试”，而是“输出不合格时重新生成”。

## 今天踩坑 / 复盘

1. 只要求“返回 JSON”通常还不够，模型有时还是会夹带解释文字，所以本地校验不能省。
2. 重试逻辑不应该塞进最底层模型调用里，否则职责会混乱。
3. Day2 先把结构化输出打稳，比一开始就做前端页面更重要。
4. Zod 这种轻量校验方式很适合前端背景的人快速上手。
