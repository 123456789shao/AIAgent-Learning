# Day3 复盘

## 今天完成了什么

1. 在 Day2 的最小 CLI 架构基础上，新建了 Day3 独立项目目录。
2. 增加了两个本地 mock 工具：天气查询和网页摘要。
3. 让模型先输出动作 JSON，再由程序决定是否执行工具。
4. 把工具结果再喂回模型，并在终端展示完整执行轨迹。

## 今天最大的收获

- Tool Calling 的核心不是“模型自己去调用工具”，而是模型负责决策，代码负责执行。
- 工具本身要小而清晰，参数越明确，模型越不容易乱用。
- 执行轨迹展示出来后，Agent 的行为会更容易理解和调试。

## 今天踩坑 / 复盘

1. Day3 不需要一上来就做多步循环 Agent，先把单次工具调用打通更重要。
2. 即使是工具调用，模型输出也不能直接信，仍然要先做 JSON 解析和 Schema 校验。
3. mock 工具已经足够让你理解 Tool Calling 本质，没必要一开始就接真实外部 API。
4. 把工具执行写在编排层，比塞进模型调用层更清楚。
