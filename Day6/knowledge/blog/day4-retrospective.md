# Day4 复盘

## 今天完成了什么

1. 在 Day3 的最小 CLI 架构基础上，新建了 Day4 独立项目目录。
2. 增加了本地 JSON 记忆文件，用来保存对话历史和用户偏好。
3. 把单轮问答升级成了交互式多轮聊天。
4. 增加了历史裁剪和摘要压缩，避免上下文无限增长。
5. 让助手可以跨会话记住用户偏好，例如语言、输出风格和技术栈。

## 今天最大的收获

- 记忆不是把所有历史都原样塞进模型，而是要分成“最近对话 + 结构化偏好 + 历史摘要”。
- 用户偏好比原样记住所有聊天内容更有价值，也更稳定。
- 上下文管理做得不好，模型成本会越来越高，效果还可能变差。

## 今天踩坑 / 复盘

1. 多轮对话如果没有裁剪策略，很容易让上下文越来越长。
2. 偏好提取不能太激进，最好只记住用户明确表达的稳定偏好。
3. 历史压缩本质上也是一次模型调用，所以同样需要结构化约束和清晰 prompt。
4. 聊天循环应该保持简单，重点先放在“记忆”而不是额外功能。

## 当前状态

已经完成 Day4 的多轮聊天最小代码骨架、本地记忆链路和偏好持久化流程。

当前你可以直接观察：
- 助手能不能记住当前会话内容
- 助手能不能跨重启保留用户偏好
- 历史过长时是否会自动裁剪和压缩
- 当前输入是否能覆盖旧偏好
